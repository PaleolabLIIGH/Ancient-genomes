### 5.5.1 Study the program
Normalization is recommended if the series datasets are different sizes—log transformation. Alternatively, within-sample proportional normalization can be implemented in R or even excel once the data 
table has been generated. 
The recommendation is to work from raw abundance values initially, (that is why we repeat the human mapping and the kraken analysis with the subsampled sequences) and examine binning power prior to 
normalization.

You can learn more about subsampling here:
```
https://github.com/lh3/seqtk
https://link.springer.com/protocol/10.1007/978-1-4939-8728-3_14
```

You already have a table with the information of how many reads you got in each of your sequences. In this case the sequence with less information had 10354090 reads after removing adapters, so I'll
normalize the rest of them with that size (except negatives, that have much less information) with the next code:
```
#Step1: Check that the future names are correct:
for file in $(find /mnt/Cromosoma/mavila/lcarrillo/HSJN-concatenado/subsampling -type f -name "*bz2*"); do
  name=$(basename $file | cut -d "_" -f 1)
  echo $name
  new_name=$(echo "$name_L001_R1-2_001.sub.fastq.collapsed")
  echo $new_name
done
#Step2: Subsampling process:
#If the name are correct then use the next one:
for file in $(find /mnt/Cromosoma/mavila/lcarrillo/HSJN-concatenado/subsampling -type f -name "*bz2*"); do
  name=$(basename $file | cut -d "_" -f 1)
  echo $name
  new_name=$(echo "$name_L001_R1-2_001.sub.fastq.collapsed")
  /mnt/Cromosoma/mavila/vvilla/scripts/seqtk/seqtk sample -s100 <(bzcat $file) 10354090 > $new_name #Remember to chance the size, in this case I used 10354090
done
```
