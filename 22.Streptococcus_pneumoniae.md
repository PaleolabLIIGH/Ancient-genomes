### 22.1 Diversity of Streptococcus pneumoniae
If you are working with *Streptococcus pneumoniae*, you’ve likely realized that it is as complex a pathogen to study as *Salmonella*. 

*S. pneumoniae* can be classified in various ways. You can classify it by serovars (referred to as serotypes in this species), where each serotype includes different substrains. Alternatively, it 
can be classified into `GPSC` (Global Pneumococcal Sequence Clusters), and there are also *Streptococcus* strains without capsular groups. 

There are many platforms with databases dedicated to this pathogen, and due to its diversity, existing phylogenies vary depending on the focus of the analysis. 

In our laboratory, when working with shotgun sequencing, it is best to use GPSC classification, as most shotgun libraries have been categorized into these capsular groups. Additionally, if you 
identified the pathogen through VF (virulence factor) analysis, it’s highly likely that you also detected capsular genes. You can verify this in tools like Artemis or IGV. 

On the GPSC website, you’ll find that there are over 70 GPSC groups, and by exploring the site, you can access hundreds of genomes available for each group, which can be used to build phylogenies. In fact, within each GPSC group, you can view phylogenies constructed with modern genomes.

```
#Website
https://www.pneumogen.net/gps/#/resources
```

Therefore, the first step to determining which group to focus on is identifying the GPSC group your ancient genome belongs to. 

There are several ways to do this, including tools like *Seroba* or the *Pathogen Informatics Training* pipeline for local machine analysis, or PubMLST for online analysis. 
```
https://sanger-pathogens.github.io/seroba/
https://github.com/sanger-pathogens/pathogen-informatics-training?tab=readme-ov-file#usage
https://pubmlst.org/
```

Here, I’ll guide you through the steps to perform this analysis using the PubMLST platform.

### 22.6 Generate the consensus sequence
Combine the reference genome and detected variants to produce a consensus sequence. This sequence reflects the specific variations found in your sample.
```
# Directory containing the VCF.gz files (adjust if necessary)
vcf_gz_dir="/mnt/Cromosoma/mavila/lcarrillo/HSJN-concatenado/Mapping_hg38/Unmapped/stre_pneumoniae"
# Reference genome
reference_genome="/mnt/Cromosoma/mavila/lcarrillo/genomes_bacteria/Streptococcus_pneumoniae.fna"
for vcf_gz_file in "$vcf_gz_dir"/*.vcf.gz; do
  base_name="${vcf_gz_file%.vcf.gz}"
  output_fasta="${base_name}.consensus.fastq"
  bcftools consensus "$vcf_gz_file" -f "$reference_genome" > "$output_fasta"
done
ls -ltrh
```
Change the extension, because the resulting file (output.consensus.fastq) contains the consensus sequence 

```
head E199-shotgun-concat_L001_R1-2_001.fastq.concat.collapsed.unmapped.fastq.bwa.sort.rmdup.uniq.consensus.fastq
# Desired output extension
output_extension=".fasta"
for input_file in "$vcf_gz_dir"/*.fastq; do
  base_name="${input_file%.*}"
  output_file="${base_name}${output_extension}"
  mv "$input_file" "$output_file"
done
```
