### 4.1 Understanding the script
Once you have your collapsed sequences, proceed to human mapping. However, before starting any process, you should carefully study what each command and program used in the script does. Here is 
the content 
for you to study:

```
### Written by Maria (maricugh@gmail.com)
###
### May 2014
### This script is based on Clean-map-call.V1.1.sh modified by Maria for
### the analysis of Cony's samples at Stanford
### Modified by VVI to make realignment
### extract unmapped reads and create bam file for mt reads

###Adapted to LIIGH cluster

#Your job name
#$-N Trim_map_script_l
#Use current working directory
#$-cwd

#Join stdout and stderr
#$-j y
# Run job through bash shell
#$-S /bin/bash
# The amount of memory requested per core (Default for many applications is 2GB).
$ -l vf=10G
#$ -pe openmp 24
#export omp_num_threads=24


#Send an email after the job has finished
#$-m e
#$-M laura-carrillo-olivas@hotmail.com

#If modules are needed, source modules environment (Do not delete the next line):
 . /etc/profile.d/modules.sh

### Arguments are
### $1=path to reference


#A report is sent to the email below

# change email to receive results there

EMAIL='laura-carrillo-olivas@hotmail.com'

####Preparing data: Cleaning and reporting stats

## $1 is the file that is taken as the first parameter of the program (e.g.in:  ./program.sh file.txt ; file.txt is saved in the $1 variable)
module load htslib/1.2.1
module load gcc/5.1.0
module load bwa/0.7.13
module load samtools/1.6
module load picard/2.6.0
module load gatk/3.4-46
module load fastqc/0.11.3

for file in *bz2

do

base=`basename $file .bz2`

base3=`basename $file .fastq.bz2`


### Mapping

bwa aln -l 500 -t 16 $1 <(bzcat $base.bz2) | bwa samse $1 -r '@RG\tID:$base\tSM:$base' - <(bzcat $base.bz2)  |samtools view -bSh - > $base.bwa.all.bam #to keep track of REAL endogenous content


echo "Total sequences" >> $base\_REPORT.txt

samtools view -c $base.bwa.all.bam >> $base\_REPORT.txt

samtools view -bh -F4 -q 30 $base.bwa.all.bam > $base.bwa.bam
samtools view -bh -f 4 $base.bwa.all.bam > $base.unmapped.bam

#Sort with samtools

samtools sort -o $base.bwa.sort.bam  $base.bwa.bam

echo "Before removing pcr clones" >> $base\_REPORT.txt

samtools view -c $base.bwa.sort.bam >> $base\_REPORT.txt

rm $base.bwa.all.bam
rm $base.bwa.bam

#Rmdup

samtools rmdup -s $base.bwa.sort.bam $base.bwa.sort.rmdup.bam
echo "After removing pcr clones" >> $base\_REPORT.txt
samtools view -c $base.bwa.sort.rmdup.bam >> $base\_REPORT.txt


#keep only uniquely mapped

samtools view -h $base.bwa.sort.rmdup.bam |grep -v 'XT:A:R'|grep -v 'XA:Z' |awk '{if($0~/X1:i:0/||$0~/^@/)print $0}' | samtools view -bS - > $base.bwa.sort.rmdup.uniq.bam

echo "After removing paralogs" >> $base\_REPORT.txt

samtools view -c $base.bwa.sort.rmdup.uniq.bam >> $base\_REPORT.txt

rm $base.bwa.sort.rmdup.bam

#Fastqc of mapped reads

fastqc -f bam $base.bwa.sort.rmdup.uniq.bam &

samtools mpileup -f $1 -C 50 $base.bwa.sort.rmdup.uniq.bam | tee $base.bwa.sort.rmdup.uniq.pileup | awk '{if($3!="*"){sum+=$4}}END{print  sum/NR "\n" NR}' >> $base\_REPORT.txt

#make index

echo "Mapped reads average length" >> $base\_REPORT.txt
samtools view $base.bwa.sort.rmdup.uniq.bam |awk '{SUM+=length($10);DIV++}END{print SUM/DIV}' >> $base\_REPORT.txt


samtools index $base.bwa.sort.rmdup.uniq.bam

##Replace readgroups with ID from sequencing

java -jar $PICARDHOME/picard.jar AddOrReplaceReadGroups \
      I=$base.bwa.sort.rmdup.uniq.bam\
      O=$base.bwa.sort.rmdup.uniq.RG.bam\
      RGLB=$base3 \
      RGPL=illumina \
      RGPU=$base3 \
      RGSM=$base3

samtools index $base.bwa.sort.rmdup.uniq.RG.bam

#Make realignment with GATK

java  -jar /cm/shared/apps/gatk/3.4-46/bin/GenomeAnalysisTK.jar \
	-T RealignerTargetCreator \
	-R $1 \
	-I $base.bwa.sort.rmdup.uniq.RG.bam \
	-o $base.intervals \

\
\

java -Xmx8G -jar /cm/shared/apps/gatk/3.4-46/bin/GenomeAnalysisTK.jar \
	-T IndelRealigner \
	-R $1 \
	-I $base.bwa.sort.rmdup.uniq.RG.bam \
	-targetIntervals $base.intervals \
	-o $base3.realign.bam \

\
\

###MD Tagging
###MD tags can be used by programs to perform e.g. SNP detection without a reference genome

#Update MD tags with samtools calmd


samtools calmd -b $base3.realign.bam $1 > $base3.realign.MD.bam

samtools index $base3.realign.MD.bam



more $base\_REPORT.txt | tr '\n' ',' > $base\_REPORT.csv


########make bam with only mt reads
samtools view -h $base3.realign.MD.bam MT |egrep 'MT|^@PG|^@H' |samtools view -hbS - > $base3.realign.rCRS.MD.bam

samtools index $base3.realign.rCRS.MD.bam

samtools view -c $base3.realign.rCRS.MD.bam > $base\_REPORT_mt.txt
more $base\_REPORT_mt.txt | tr '\n' ',' > $base\_REPORT_mt.csv

mkdir Mapping_mt
mv *rCRS* Mapping_mt/
mv $base\_REPORT_mt.txt Mapping_mt/
mv $base\_REPORT_mt.csv Mapping_mt/

# unmapped bam to fastq

module load bedtools/2.26
bedtools bamtofastq -i $base.unmapped.bam -fq $base.unmapped.fastq
rm $base.unmapped.bam

bzip2 $base.unmapped.fastq

mkdir Unmapped
mv $base.unmapped.fastq.bz2 Unmapped/


cd Mapping_mt
mkdir Schmutzi
mkdir Contammix
cp *rCRS*.ba* Schmutzi/
cp *rCRS*.ba* Contammix/
cd ../



done


exit;
```

### 4.2 Modify the script
As you can see, the only change you really need to make is to update your email; the rest remains unchanged.

The first step is to copy the script to your scripts directory:

```
cp /mnt/Cromosoma/mavila/backup/lcarrillo/scripts/Mapping_hg38-rCRS_seed_dic2020-lco.sh /mnt/Cromosoma/mavila/lcarrillo/scripts/
```

### 4.3 Prepare the files for mapping
Next, go to your working directory and create a folder for mapping where you will place symbolic links to the collapsed files:

```
cd /mnt/Cromosoma/mavila/lcarrillo/capturas_salmonella_myco
mkdir Mapping_hg38
for file in $(find /mnt/Cromosoma/mavila/lcarrillo/capturas_salmonella_myco -type f -name "*fastq.collapsed.bz2"); do
    ln -s $file Mapping_hg38
done
```

You can check that the symbolic links were created:

```
ls Mapping_hg38
```

NOTE 1: If the links appear in red, they were not copied correctly and the process needs to be repeated.

NOTE 2: If the links appear in dark blue instead of light blue, symbolic hard links were created instead of soft links for some reason. In this case, do not delete them because deleting them will also 
remove your original file. Just continue with the process.

### 4.4 Run the mapping
We now have everything we need. Next, switch to the Mapping_hg38 directory and run the script, specifying the path to the reference genome like this:

```
cd Mapping_hg38/
qsub /mnt/Cromosoma/mavila/lcarrillo/scripts/Mapping_hg38-rCRS_seed_dic2020-lco.sh /panfs/pan5/mavila/vvilla/genome_ref/human/GRCh38.p13/GRCh38.p13_chr-only/GRCh38.p13.rCRS.fasta
qstat
```
