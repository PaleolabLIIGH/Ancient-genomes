### 29.1 Verify Chromosome Identifiers in the Original VCF
The VCF header contains information about the chromosomes. You can search for lines starting with ##contig to see how the chromosomes are defined:
```
cd /panfs/pan5/mavila/lcarrillo/biobank_mx/23-02-14-MXB50GENOMASCOMPLETOS/tosharemxb50wgs
grep '##contig' mexican_50_autosomes.vcf 
```
This will display something like:
```
##contig=<ID=chr1,length=248956422>
##contig=<ID=chr2,length=242193529>
...
##contig=<ID=chr23,length=156040895>
##contig=<ID=chr24,length=57227415>
##contig=<ID=chr26,length=16569>
```
If you see chr23, chr24, and chr26, then the sex chromosomes and mitochondrial chromosome are encoded that way.
Or you can inspect the first variant lines to see how the chromosomes are labeled:
```
grep -v '^#' mexican_50_autosomes.vcf | cut -f1 | sort | uniq -c
```
This will output something like:
```
1123456 chr1
 987654 chr2
  12345 chr23
   5678 chr24
     10 chr26
```
If you see chr23, chr24, and chr26, then the sex chromosomes and mitochondrial chromosome are encoded that way.

### 29.2 Change Chromosome Identifiers in the VCF (if necessary)
If you confirm that the chromosomes are encoded as chr23, chr24, and chr26, you can modify them directly in the VCF file before proceeding with further analysis. Here’s how:
```
sed -i 's/\bchr23\b/chrX/g' mexican_50_autosomes.vcf
sed -i 's/\bchr24\b/chrY/g' mexican_50_autosomes.vcf
sed -i 's/\bchr26\b/chrM/g' mexican_50_autosomes.vcf
```
After making the changes, you can verify the chromosome identifiers again:
```
grep '##contig' mexican_50_autosomes.vcf
grep -v '^#' mexican_50_autosomes.vcf | cut -f1 | sort | uniq -c
```
Now, you should see chrX, chrY, and chrM instead of chr23, chr24, and chr26.

### 29.3 Convert the VCF File to PLINK Format
```
screen -S vcf_to_plink
qlogin -pe openmp 12
cd /panfs/pan5/mavila/lcarrillo/biobank_mx/23-02-14-MXB50GENOMASCOMPLETOS/tosharemxb50wgs
module load plink2/1.90beta3
plink --vcf mexican_50_autosomes.vcf --make-bed --out MexBio_50autosomes
```
This command converts a VCF file `mexican_50_autosomes.vcf` into PLINK format, generating .bed, .bim, and .fam files.
    MexBio_50autosomes.bed: Binary genotype data.
    MexBio_50autosomes.bim: Variant information.
    MexBio_50autosomes.fam: Individual information.
But you will obtain this error:
```
Line 130: INFO field contains an invalid value (NaN).  
Invalid alternative alleles: Some lines contain `*` in the ALT field.  
```
Replace `NaN` with `.`
```
sed 's/NaN/./g' mexican_50_autosomes.vcf > fixed_mexican_50_autosomes.vcf
```
Hypothetical example:
```
#Before:
#CHROM  POS     ID      REF  ALT  QUAL  FILTER  INFO
1       15274   rs123   A    T    100   PASS    NaN
#After:
#CHROM  POS     ID      REF  ALT  QUAL  FILTER  INFO
1       15274   rs123   A    T    100   PASS    .
```
Remove Records with Symbolic Alleles (*)
```
module load bcftools/1.17
bcftools view -e 'ALT~"\*"' fixed_mexican_50_autosomes.vcf -o fixed_mexican_50_autosomes_nosym.vcf
```
This command uses bcftools to remove lines where the ALT field contains * (an invalid symbolic allele).
Hypothetical example:
```
#Before:
#CHROM  POS     ID      REF  ALT  QUAL  FILTER  INFO
1       15274   rs123   A    T    100   PASS    NaN
1       30923   rs456   C    *    99    PASS    .
#After:
#CHROM  POS     ID      REF  ALT  QUAL  FILTER  INFO
1       15274   rs123   A    T    100   PASS    .
```
Run PLINK Again with the Fixed VCF File
```
plink --vcf fixed_mexican_50_autosomes_nosym.vcf --make-bed --out MexBio_50autosomes
```
Now that the VCF file is corrected, PLINK can successfully generate the PLINK-formatted files.
```
#Output:
11,236,529 variants and 50 individuals pass filters and QC.
```
Remember that MexBio_50autosomes.bed is binary, but the other ones are going to have this structure:
```
#MexBio_50autosomes.bim:
    1  rs123  0  15274  A  T

#MexBio_50autosomes.fam:
    IND1   IND1  0  0  1  -9
    IND2   IND2  0  0  2  -9
```
### 29.4 Changing SNP IDs from . to chr_pos in the .bim File
The .bim file contains variant information, with the second column representing the SNP ID.
If SNPs have missing IDs (.), we replace them with a combination of chr_pos (chromosome and position).
```
awk '{ $2 = $1 "_" $4; print }' MexBio_50autosomes.bim > temp && mv temp MexBio_50autosomes.bim
```
Hypothetical example:
```
#Before:
1  .  0  15274  A  T
1  rs123  0  30923  C  G
#After:
1  1_15274  0  15274  A  T
1  rs123  0  30923  C  G
```
This ensures that previously missing SNP IDs (.) are now replaced with their chr_pos format.

### 29.5 Converting Files to PED/MAP Format and Back to PLINK to Fix Corruption
Sometimes, PLINK files can become corrupted or contain formatting inconsistencies. A common workaround is converting them to PED/MAP format and then back to BED/BIM/FAM. This ensures that 
the files are properly formatted and compatible with downstream analyses.
```
plink --bfile MexBio_50autosomes --allow-extra-chr --recode --out temp
```
This will generate the files temp.ped and temp.map
Hypothetical example:
```
#MexBio_50autosomes.bim:
1  rs123  0  15274  A  T
1  rs456  0  30923  C  G
#MexBio_50autosomes.fam:
IND1  IND1  0  0  1  -9
IND2  IND2  0  0  2  -9
#temp.ped:
IND1  IND1  0  0  1  -9  A A  C C
IND2  IND2  0  0  2  -9  T T  G G
Each line represent an individual
Family ID, Individual ID, Parental IDs (0 = unknown), Sex (1 = male, 2 = female), Phenotype (1/2 or -9 for missing), followed by genotypes.
#temp.map:
1  rs123  0  15274
1  rs456  0  30923
Each line represent a SNP
Chromosome, SNP ID, Genetic Distance (0 if unknown), Physical Position.
```
Back to PLINK
```
plink --file temp --make-bed --allow-extra-chr --out MexBio_50autosomes_IDs
```
Final Output (Corrected PLINK Files):
```
    MexBio_50autosomes_IDs.bed → Binary genotype data.
    MexBio_50autosomes_IDs.bim → SNP information.
    MexBio_50autosomes_IDs.fam → Individual information.
```
### 29.6 Changing Individual IDs to 'MXB_...' to Avoid Duplicates
Sometimes, individual IDs in a .fam file might be duplicated or need a standardized prefix for better organization. The following command prepends "MXB_" to each individual ID in the .fam 
file.
```
awk '{$2 = "MXB_" $2; print}' MexBio_50autosomes_IDs.fam > temp && mv temp MexBio_50autosomes_IDs.fam
```
### 29.7 Converting Files to PED/MAP Format and Back to PLINK Format
This process is often used to clean corrupted files or resolve formatting issues that might cause problems in downstream analyses. Converting .bed/.bim/.fam files to .ped/.map and then 
back to .bed/.bim/.fam ensures data integrity.
```
plink --bfile MexBio_50autosomes_IDs --allow-extra-chr --recode --out temp
plink --file temp --make-bed --allow-extra-chr --out MexBio_50autosomes_IDs2
```
 This will generate the following files:
 ```
    MexBio_50autosomes_IDs2.bed
    MexBio_50autosomes_IDs2.bim
    MexBio_50autosomes_IDs2.fam
 ```
### 29.8 Converting Coordinates from hg19 to hg38
Convert the .bim file (PLINK format) into a .bed file (BED format) for use in the LiftOver tool, which translates genomic coordinates from one reference version (hg19) to another (hg38).
```
awk '{print "chr"$1, $4-1, $4, $2, $5, $6}' MexBio_50autosomes_IDs2.bim > MXB_50_toLift.bed
```
The BED format consists of six columns:
```
    Chromosome (e.g., chr1, chr2).
    Start position (0-based).
    End position (1-based).
    Variant name (SNP ID).
    Reference allele.
    Alternate allele.
```
Hypothetical Example
```
#MexBio_50autosomes_IDs2.bim
1  rs123  0  15274  A  T  
1  rs456  0  30923  C  G  
#MXB_50_toLift.bed
chr1 15273 15274 rs123 A T  
chr1 30922 30923 rs456 C G  
```
This file is now ready for LiftOver conversion.

### 29.9 Execute LiftOver
```
module load liftover/liftover
export LD_LIBRARY_PATH=/mnt/Cromosoma/mavila/frodriguez/Afromexico/Updated_positions/Ancient_HSJN/LiftOver/lib/x86_64-linux-gnu:$LD_LIBRARY_PATH
liftOver MXB_50_toLift.bed /mnt/Cromosoma/mavila/frodriguez/Afromexico/Updated_positions/LiftOver/UCSC/hg19ToHg38.over.chain.gz Lifted_MXB_50_Hg38.bed unlifted_MXB_50_Hg38.bed
```
The chain file `hg19ToHg38.over.chain.gz` maps coordinates from hg19 to hg38. This file is provided by UCSC and contains rules for converting coordinates between the two genome versions.
This code will generate the files `Lifted_MXB_50_Hg38.bed` (Output file containing the successfully converted coordinates in hg38.) and `unlifted_MXB_50_Hg38.bed` (Output file containing coordinates that could not be converted (e.g., due to unclear mapping between hg19 and hg38).
Hypothetical Example
```
#MXB_50_toLift.bed
chr1 15273 15274 rs123 A T  
chr1 30922 30923 rs456 C G
#hg19ToHg38.over.chain.gz)
##This is a binary file containing rules for converting hg19 to hg38 coordinates. It is not directly readable, but conceptually, it works like a mapping table:
chr1 15273 15274 -> chr1 16000 16001  
chr1 30922 30923 -> chr1 35000 35001  
#Lifted_MXB_50_Hg38.bed
chr1 16000 16001 rs123 A T  
chr1 35000 35001 rs456 C G  
#unlifted_MXB_50_Hg38.bed
chr1 50000 50001 rs789 G A  
```
