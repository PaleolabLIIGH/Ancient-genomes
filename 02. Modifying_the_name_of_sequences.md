### 2.1 Introduction to regex
Depending on where your files were sequenced, the structure of their names may vary. However, before starting the analyses, it's beneficial to rename them for future loops in subsequent analyses.

You can choose to rename each file individually, but this becomes tedious when you have multiple sequences. Therefore, here we explain the basics of regular expressions for renaming your files using a loop.

NOTE: Keep in mind that you should understand the structure of the original names of your sequences to create the desired structure.

In this series of sequences, certain patterns exist:

```
ls
#Output:
021-H238-WISC_S1_L001-4_R1_001.fastq.bz2     091-H337bis-WISC_S3_L001-4_R2_001.fastq.bz2
021-H238-WISC_S1_L001-4_R2_001.fastq.bz2     099-H402-WISC_S4_L001-4_R1_001.fastq.bz2
041-H302-WISC_S2_L001-4_R1_001.fastq.bz2     099-H402-WISC_S4_L001-4_R2_001.fastq.bz2
041-H302-WISC_S2_L001-4_R2_001.fastq.bz2     109-H402c-WISC_S5_L001-4_R1_001.fastq.bz2
091-H337bis-WISC_S3_L001-4_R1_001.fastq.bz2  109-H402c-WISC_S5_L001-4_R2_001.fastq.bz2
```
All filenames start with a series of three numbers, for example, `021`, which we will call group 1. This can be represented using regular expressions (regex) as `^[0-9]`, which symbolizes a group 
(not necessarily just three numbers, but any number of digits together) ranging from 0 to 9. The `^` symbol indicates that the expression starts with these numbers.

These three numbers are followed by a hyphen. To represent this combination with regex, it would be `[0-9]+-`.

Next, there is a letter "H" and another series of three numbers "238". In regex, to represent both uppercase and lowercase letters along with numbers, we use `[A-Za-z0-9]`. Therefore, combining the 
previous expressions, it would be `^([0-9]+-[A-Za-z0-9])`. We'll call it group 1. Each group is defined by parentheses `()`.

This is followed by another hyphen and then four letters `WISC`. Since "WISC" fits the previous pattern but appears after a hyphen, we simply include the hyphen in group 1's expression, resulting in:
`^([0-9]+-[A-Za-z0-9-]+)`.

Next, there is an underscore and a letter `S` followed by a number `1`. To represent this in regex, it would be `^([0-9]+-[A-Za-z0-9-]+)_S[0-9]`.

All sequences share a common pattern after `S#`, so we add it as is: `^([0-9]+-[A-Za-z0-9-]+)_S[0-9]+_L001-4_R`.

Since some have "R1" or "R2", that number will be group 2: `^([0-9]+-[A-Za-z0-9-]+)_S[0-9]+_L001-4_R([0-9]+)`.

Finally, the pattern ends the same way in all sequences, so we add it as is: `^([0-9]+-[A-Za-z0-9-]+)_S[0-9]+_L001-4_R([0-9]+)_001.fastq.bz2`.

This regex expression represents all the sequences.

Now that we know which regex to use, we test a for loop to rename the files:

```
for file in *_S*_L001-4_R*_001.fastq.bz2; do
     new_name=$(echo "$file" | sed -E 's/^([0-9]+-[A-Za-z0-9-]+)_S[0-9]+_L001-4_R([0-9]+)_001.fastq.bz2/\1_L001-4_R\2_001.fastq.bz2/')
     echo "$new_name"
 done
```
Here, `sed -E` is the command we use to apply the regex substitution. The first `\1` represents group 1, followed by the pattern `_L001-4_R`, and then group 2, `\2`, followed by the pattern 
`_001.fastq.bz2`.

If the name is correct, you got it! If not, try again. Maybe a number or letter was missed in your regex. Once you have the correct regex, rename the files with the next code.

```
for file in *_S*_L001-4_R*_001.fastq.bz2; do
     new_name=$(echo "$file" | sed -E 's/^([0-9]+-[A-Za-z0-9-]+)_S[0-9]+_L001-4_R([0-9]+)_001.fastq.bz2/\1_L001-4_R\2_001.fastq.bz2/')
     mv "$file" "$new_name"
 done
```
