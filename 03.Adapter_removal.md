### Method 2
In your work directory copy the next script:
```
cd /mnt/data/mavila/lcarrillo/Streptococcus_pneumoniae_concatenado_paper/GPSC105

```
This is the script:
```
### Written by Viridiana Villa
### Modified by Laura Carrillo
### Parallel version for Fenix cluster

#SBATCH --job-name=AdapterremovalLCO
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=laura-carrillo-olivas@hotmail.com
#SBATCH --cpus-per-task=20
#SBATCH --mem-per-cpu=3G
#SBATCH --ntasks=1
#SBATCH --nodes=1

source /etc/profile.d/modules.sh
module load adapterremoval/1.5.4

wdir="$PWD"

fbz1="_1.fastq.bz2"
fbz2="_2.fastq.bz2"

THREADS_PER_SAMPLE=10
MAX_JOBS=$(($SLURM_CPUS_PER_TASK / THREADS_PER_SAMPLE))

echo "CPUs available: $SLURM_CPUS_PER_TASK"
echo "Threads per sample: $THREADS_PER_SAMPLE"
echo "Samples running in parallel: $MAX_JOBS"

for file in $(find $wdir -type f -name "*$fbz1"); do
(
    nm=$(basename "$file" _1.fastq.bz2)

    echo "Processing sample: $nm"

    rem1="${nm}${fbz1}"
    rem2="${nm}${fbz2}"
    fq3="${nm}_1-2.fastq"

    /mnt/data/mavila/lcarrillo/Adapter-removal/AdapterRemoval2/adapterremoval-2.3.1/build/AdapterRemoval \
        --file1 $rem1 \
        --file2 $rem2 \
        --basename $fq3 \
        --threads $THREADS_PER_SAMPLE \
        --trimns \
        --trimqualities \
        --qualitybase 33 \
        --minlength 30 \
        --collapse \
        --bzip2 \
        --mate-separator .

    report="${nm}_adapter_collapsed.txt"

    echo "Reads" > $report
    bzcat "${nm}_1.fastq.bz2" | wc -l >> $report

    echo "Collapsed after adapterremoval" >> $report
    bzcat "${fq3}.collapsed.bz2" | wc -l >> $report

    coll="${fq3}.collapsed.bz2"
    colltrun="${fq3}.collapsed.truncated.bz2"
    disc="${fq3}.discarded.bz2"
    pair1="${fq3}.pair1.truncated.bz2"
    pair2="${fq3}.pair2.truncated.bz2"
    sett="${fq3}.settings"
    sing="${fq3}.singleton.truncated.bz2"

    fastq_dir="${nm}_fastq"

    mkdir -p $fastq_dir

    mv $coll $fastq_dir/
    mv $colltrun $fastq_dir/
    mv $disc $fastq_dir/
    mv $pair1 $fastq_dir/
    mv $pair2 $fastq_dir/
    mv $sett $fastq_dir/
    mv $sing $fastq_dir/

) &

if [[ $(jobs -r -p | wc -l) -ge $MAX_JOBS ]]; then
    wait -n
fi

done

```

However, checking each file manually can be very exhausting if you have many sequences to analyze. Therefore, you can use the script get-table-adapter.sh 
to obtain the information for all sequences simultaneously.

The first step is to copy the script to your scripts folder:
```
cp /mnt/Cromosoma/mavila/backup/lcarrillo/scripts/get-table-adapter.sh /mnt/Cromosoma/mavila/lcarrillo/scripts #Modify destine
```

The information of the script is here:

```
### Written by Laura Carrillo Olivas
###
### Aug 2023
### This script is to get the results of each individual after the adapterremoval script in a single table

###Adapted to LIIGH cluster

#Your job name
#$-N table-adapter
#Use current working directory
#$-cwd

#Join stdout and stderr
#$-j y
# Run job through bash shell
#$-S /bin/bash

#Send an email after the job has finished
#$-m e
#$-M laura-carrillo-olivas@hotmail.com #Modify e-mail

#If modules are needed, source modules environment (Do not delete the next line):
 . /etc/profile.d/modules.sh

### Arguments are
### $1=path to reference


#A report is sent to the email below

# change email to receive results there

EMAIL='laura-carrillo-olivas@hotmail.com' #Modify e-mail

# Encabezado de la tabla
echo -e "Archivo\tReads por 4\tCollapsed after adapterremoval"

# Recorrer todos los archivos _collapsed.txt y extraer la información
for file in *_adapter_collapsed.txt; do
    archivo=$(basename "$file" -_adapter_collapsed.txt)  # Obtener el nombre del archivo sin extensión

    reads_por_4=$(awk '/Reads/ { getline; print $1 }' "$file")
    collapsed_after_adapterremoval=$(awk '/Collapsed after adapterremoval/ { getline; print $1 }' "$file")

    # Imprimir la línea de la tabla
    echo -e "$archivo\t$reads_por_4\t$collapsed_after_adapterremoval"
done
```

Now, run it as follows:
```
bash /mnt/Cromosoma/mavila/lcarrillo/scripts/get-table-adapter.sh
```
